Deployment process

All Front-End project using Vercel as a hosting.

To deploy the application on Vercel, all we need to do is push the code to GitHub. Each time we push or merge code into the main branch, deployment to production occurs.

Each time we create a new branch and push it to GitHub, Vercel creates a test version of the application (with the code from the latest branch). Everything happens automatically; all you need to do is push the code.

For each branch, you can choose a specific environment to avoid interference with the production environment.

For testing Sidebar, you need go to the AppScript add-on application and following AppScript instruction. Open add-on in the spreadsheet.

Code Highlights:

Deploy app to the production

git push origin main

Deploy app to the test Enviroment

git push origin new-feature-branch
