Environments

This documentation outlines the essential environment variables required for the seamless functioning of the application. These variables include:

GOOGLE_CLIENT_ID: Purpose: Google OAuth client identifier for authentication purposes. Example: Contact team, to get this key GOOGLE_CLIENT_SECRET: Purpose: Google OAuth client secret for secure authentication. Example: Contact team, to get this key NEXT_AUTH_SECRET: Purpose: A secret key used by Next.js for session authentication. Example: Contact team, to get this key API_URL: Purpose: URL endpoint for the primary API service. Example: https://rdfp-engine-7o7tsdjkha-uc.a.run.app NEXT_PUBLIC_INTERNAL_API_URL: Purpose: Internal API URL, often used during development. Example: http://localhost:3000 API_KEY: Purpose: A key for API authentication or access control. Example: test INITIAL_SHEET_ID: Purpose: Identifier for the initial Google Sheet used by the application. Example: 1EevD-XsrmD6IqGPd62DXG8m0l4l4gESy4IYn3BKbdD8 SLACK_WEBHOOK: Purpose: URL for the Slack webhook, enabling integration with Slack. Example: Contact team, to get this key

Ensure that these environment variables are properly configured to ensure the correct operation of the application across different environments, such as development, testing, and production.
