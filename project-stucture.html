Project Structure

  • public - The public directory in a Next.js project is a special directory that allows you to serve static assets like images, fonts, and other files directly. Any file you place in the public directory will be served at the root level of your application.

    • fonts - folder include all project fonts

  • src - The src directory in Next.js - is a root directory, which include entire project code.

    • api - This folder include internalClient and externalClient classes, which allow communicate with internal (Next.JS) and external (API) servers

    • utils - This folder includes helper function;

    • types - This folder inlcudes types and interfaces for TypeScript

    • enums - This folder includes enums

    • constants - This folder includes two important files for the chat

    • services - This folder include functions, that make calls to the external API

    • messages.ts - This file includes an object with all messages their sequences. Field component store a component from components/messages. Also next field indicates which component will be the next in the chat sequences.

    • selectionOptions - This file includes objects, with options for select in the chat

    • Components - This folder include two folder with server and client components (please read Next.js documentation about difference betwen those types of components)

      1. Clients - This folder includes client component (most of them are UI component, header, button and etc.)

      2. Servers - This folder includes server components (with business logic)

      P.S. It could be some confusing, that client components are in the Server folder, probably it's a mistake of developer.

  • app - root folder with routes, where each page is a route

    • api - internal API, which is used for calling external API

    • Folder with pages - routes (read Next.js doc for better understanding)
